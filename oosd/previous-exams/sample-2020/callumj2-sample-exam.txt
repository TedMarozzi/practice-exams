Abstraction is the one of the principles of object oriented programming. It is the idea that all information, data and
coding relating to a class should be hidden from other classes and relevant pieces of information should only be 
accessible through the classes public methods. Abstraction is achieved in object oriented programming mainly through 
privacy access modifiers.

The term immutable refers in this context to the inability of a class to be changed by itself or other classes. This is
achieved in java by setting all the classes variables to private and final and only allowing viewing access via getter
functions as well as not creating any setter functions.

The Factory Method design pattern solves the problem of creating multiple instances of subclass belonging to a common
super class by abstracting the class creation coding away from the main body of code and into a 'factory' class. The
components of the factory design pattern are:
- The Client: The class which makes a specific spawn request of the factory
- The Factory: Which receives specific spawn requests and handles the instantiation of new objects.
- The Super Class: The general type(s) of class the factory can return:
- Specific SUbclasses: Multiple specific subclasses which can be created when the factory is called to create a new
object.
